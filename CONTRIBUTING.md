# Contributing

This curriculum is an instructional sequence. Contributions should preserve the sequence, the tone, and the evidence model.

## Tone

Use a firm, humane voice. Name gaps clearly without making learners feel scolded. Prefer practical language over dramatic language.

## Instructional standard

Every learner-facing module should include:

- clear goal
- concept before tool
- concrete example
- practice task
- verification step
- common confusion
- relevant spine prompts

## Spine prompts by track

When adding spine prompts to a module, refer to the [spine-by-track matrix](../spines/spine-by-track-matrix.md) to know which spines are primary (gate-blocking) vs. secondary (practice encouraged).

- **Primary spines** for the track should appear prominently in gates and key modules.
- **Secondary spines** can appear where relevant but are not required for every module.
- Do not add prompts for all eight spines to every module. That creates checkbox fatigue.

## Review checklist

- Does the lesson assume a prerequisite that has not been taught?
- Does it give the learner a way to verify progress?
- Does it preserve the AI policy for the track?
- Does it produce evidence that can be used at a gate?
- Do the spine prompts match the primary/secondary weighting for this track?
