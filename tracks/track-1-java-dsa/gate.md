# Track 1 Gate — Java and Data Structures

This gate verifies that you are ready to leave Track 1 and begin reading existing codebases in Track 2.

The goal is not perfect code. The goal is intentional code: you can choose data structures for reasons, analyze their cost, and explain your decisions.

## Required artifact

Submit your Track 1 project.

It should include:

- working Java code that solves the stated problem
- a README explaining the problem, your approach, and your data structure choices
- at least one test class
- a self-evaluation note

## Correctness requirements

You must be able to:

- run the code and produce correct output
- demonstrate that edge cases are handled
- explain what each class and method does

## Explanation requirements

You must explain:

- why you chose the data structures you used
- what the complexity of your solution is and why
- what would change if the input size grew significantly
- what a specific error message in your code means
- how you found and fixed at least one bug

## Debugging and revision evidence

Provide one debugging note that includes:

- what you expected
- what happened instead
- what you checked first
- what hypothesis you tested
- what fixed it or what you learned

## Spine evidence

### Big-O and data structures

Name the primary data structure in your solution. Explain why it fits this problem better than at least one alternative. State the time and space complexity of the critical operation.

### Discipline of Doubt

Name one edge case your code handles. What breaks if the input is empty, null, very large, or repeated?

### Debugging

Walk through one error you encountered. What did the error message say? What did you check? What was the actual cause?

### Testing maturity

Show at least one test. Explain what behavior it proves and what it does not cover.

### Self-evaluation

What is the strongest part of your solution? What would you improve with more time?

### Technical communication

Show your README. Can a reader understand the problem, your approach, and how to run your code without asking you questions?

### AI collaboration

Confirm that no AI was used for Track 1 curriculum work.

## Outcome

### Pass

You can write working Java code, choose data structures intentionally, explain the complexity, and describe how you debugged the work.

### Revise

You are close, but must strengthen missing evidence: an unexplained data structure choice, missing complexity analysis, absent test, or incomplete debugging note.

### Repeat

You should revisit Track 1 if you cannot write working Java, cannot explain why you used a data structure, or cannot read a stack trace to find a bug.
