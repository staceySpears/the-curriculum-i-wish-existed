# Track 3 — Backend

Track 3 is where the learner builds server-side systems.

This track introduces HTTP, REST APIs, backend frameworks, and database basics. The goal is to build systems that are correct, testable, and explainable — not just working.

## What this track teaches

- HTTP: requests, responses, status codes, headers
- REST API design: resources, routes, methods, contracts
- Backend framework basics (Spring Boot or equivalent)
- Connecting to a database and performing basic operations
- Writing and running tests for API behavior
- Documenting contracts and endpoints
- Debugging HTTP and server errors
- Error handling and input validation

## AI policy

AI may be used as an explainer in Track 3.

You may use AI to explain concepts, error types, framework behavior, or documentation.

**Restriction:** You may not ask AI to write code for you, fix your assignment, or debug your specific implementation. You may ask AI to explain an error category, a framework concept, or a general debugging approach. Generic examples are allowed. Assignment-specific implementation is not.

See `../../AI_POLICY.md` and `../../spines/ai-collaboration.md`.

## Track 3 definition of done

Track 3 is complete when you can:

- design and implement a REST API
- connect to a database and perform basic operations
- write tests that verify API behavior
- handle errors and invalid input explicitly
- document the API contract in a README
- explain how a request flows through your system
- debug HTTP errors using logs and status codes
- pass the Track 3 gate
