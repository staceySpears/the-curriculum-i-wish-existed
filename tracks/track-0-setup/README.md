# Track 0 — Setup and Mental Models

Track 0 is for learners starting from genuine zero.

This track does not assume you already know what a terminal is, how files and folders work, how code runs, what Git does, or how to recover when setup breaks.

The goal is to make the development environment understandable enough that future tracks can focus on programming instead of hidden setup confusion.

## What this track teaches

Track 0 teaches the basic substrate of software work:

- what code is
- what a program is
- how files, folders, and paths work
- how to use the terminal without panic
- how to install and verify tools
- how to use a code editor
- how to run a simple program
- how to use Git at a beginner level
- how to read error messages literally
- how to make a first reproducible submission

## AI policy

AI is not allowed for Track 0 curriculum work.

Do not use AI to explain your exact task, write commands for you, fix your setup, generate reflections, or complete the project.

Use the curriculum docs, official documentation, human-written resources, the community, and human help if available.

The reason is substrate. Track 0 is where you build the mental model that makes later AI use safe.

See `../../AI_POLICY.md`.

## Exercises

Track 0 exercises are simple but important. They verify that you can navigate, run commands, edit files, and explain what happened.

See `exercises/README.md`.

## Project

The Track 0 project is a setup-validation project.

You will create or modify a small repository, run a simple program, make a small change, verify the change, commit it with Git, and write a short reflection.

See `project/README.md`.

## Gate

The Track 0 gate proves that you can use your environment without treating it as magic.

You pass when you can set up, run, edit, verify, commit, and explain a small software task.

See `gate.md`.

## Track 0 definition of done

Track 0 is complete when you can:

- navigate files and folders
- use basic terminal commands
- open and edit code in an editor
- run a simple program
- read an error message without panicking
- use basic Git commands
- explain what broke during setup and how you resolved it
- submit the Track 0 project
- pass the Track 0 gate
