# Track 0 Project — Setup Validation

The Track 0 project proves that your environment works and that you can complete a tiny software workflow.

This is not meant to be impressive. It is meant to be reproducible.

## Project goal

Create a small repository that shows you can:

- organize files
- edit text or code
- run a tiny program or command
- verify output
- use Git
- document what happened
- explain one setup or debugging issue

## Required files

Your project should include:

```text
track-0-validation/
  README.md
  notes/
    setup-notes.md
    debugging-note.md
  src/
    [tiny program or starter file]
```

The exact program can vary by environment, but it must produce visible output when run.

## README requirements

Your README should explain:

- what this project is
- what command runs it
- what output to expect
- what files are included
- what setup issue or confusion you encountered
- how you resolved or investigated that issue

## Git requirements

Your repository should include at least three meaningful commits:

1. Create initial project structure
2. Add runnable program or task
3. Add notes and reflection

More commits are fine.

## Reflection prompts

Answer these in your README or a separate reflection section:

- What part of setup felt most confusing?
- What error or unexpected result did you encounter?
- How did you investigate it?
- What command or concept feels clearer now?
- What would you check first if something similar broke again?

## Done means

The project is done when another person could open the repository, read the README, run the command, see the expected output, and understand what you learned.
