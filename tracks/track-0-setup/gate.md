# Track 0 Gate — Setup and Mental Models

This gate verifies that you are ready to leave Track 0 and begin programming fundamentals.

The goal is not perfection. The goal is environment readiness and basic mental model stability.

## Required artifact

Submit your Track 0 setup-validation project.

It should include:

- organized project files
- a runnable tiny program or command
- a README
- setup notes
- a debugging note
- at least three meaningful Git commits

## Correctness requirements

You must be able to:

- navigate to the project folder
- list the project files
- run the program or command
- produce the expected output
- show a clean Git status after committing
- explain what each required file is for

## Explanation requirements

You must explain:

- what code is
- what a file and folder are
- what a path is
- what the terminal does
- what your editor does
- what command runs your project
- what Git tracked in your repository
- what a commit represents

## Debugging and revision evidence

Provide one debugging note that includes:

- what you expected
- what happened instead
- what you checked first
- what hypothesis you tested
- what fixed it or what you learned

## Spine evidence

### Discipline of Doubt

Name one setup assumption that turned out to matter.

### Debugging

Walk through one problem using the debugging loop.

### Code reading

Point to one file in your project and explain what it contains.

### Testing maturity

Explain how you manually verified that the project works.

### Self-evaluation

Name what feels solid and what still feels fragile.

### Technical communication

Show how your README helps another person run the project.

### AI collaboration

Confirm that no AI was used for Track 0 curriculum work.

## Outcome

Choose one.

### Pass

You can use your environment, run a tiny task, commit work, and explain the basic workflow.

### Revise

You are close, but must fix missing evidence, unclear explanation, broken commands, or incomplete Git history.

### Repeat

You should revisit Track 0 if you cannot navigate the project, run the task, explain the files, or recover from basic setup confusion.
