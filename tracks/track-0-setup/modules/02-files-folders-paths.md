# Module 2 — Files, Folders, and Paths

## Goal

Understand how files and folders are organized and how paths describe locations.

## Core idea

A file stores information.

A folder contains files or other folders.

A path describes where something is located.

Developers constantly move between files and folders. If you do not know where you are, commands become confusing quickly.

## Important terms

A **file** is a named piece of stored information.

A **folder** or **directory** is a container for files and other folders.

A **path** is the route to a file or folder.

A **current directory** is the folder your terminal is currently operating inside.

## Absolute and relative paths

An absolute path starts from the root of the file system.

A relative path starts from where you are now.

You do not need to master every path detail yet. You do need to build the habit of asking:

> Where am I right now?

## Practice

Create this folder structure:

```text
track-0-practice/
  notes/
    setup-notes.txt
  sandbox/
    hello.txt
```

Write one sentence in each file.

Then answer:

- What folder contains `setup-notes.txt`?
- What folder contains `hello.txt`?
- What is the path from `track-0-practice` to `hello.txt`?
