# Module 01 — What Is Code?

## Goal

Understand what code is, what a program is, and what it means to run software.

## Core idea

Code is written instructions for a computer.

A program is code that can be run to make the computer do something.

The computer does not understand intention. It follows instructions exactly, including instructions that are incomplete, incorrect, or in the wrong order.

That is why programming requires precision.

## Mental model

When you write code, you are usually working with three things:

1. A file that contains instructions.
2. A tool that reads or runs those instructions.
3. Output that shows what happened.

For example, a tiny program might print a message. The code is the instruction. The tool runs it. The terminal shows the output.

## Common beginner confusion

Many beginners think the computer understands what they meant.

It does not.

If a command fails, it does not mean you are stupid. It means the computer received instructions it could not follow in the current context.

Your job is to learn how to inspect that context.

## Practice

Create a text file called `hello.txt` and write one sentence inside it:

```text
Hello from Track 0.
```

Then answer:

- Where is the file saved?
- What is the exact filename?
- What program did you use to edit it?
- How do you know the file exists?

## Spine prompts

### Discipline of Doubt

What assumption are you making about where the file was saved?

### Debugging

If you cannot find the file, what is the first thing you should check?

### Technical communication

Explain the difference between code, a file, and a program in your own words.
