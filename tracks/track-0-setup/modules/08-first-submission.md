# Module 08 — First Submission

## Goal

Make a small, complete, reproducible submission.

## Core idea

A submission is not just a file.

A good submission lets someone else understand what you changed, how to run it, what evidence shows it works, and what you learned.

Track 0 ends by proving that you can complete a tiny software workflow from start to finish.

## Submission checklist

Before submitting, confirm:

- the project folder is organized
- the program or task runs
- the expected output appears
- files are saved
- Git status is clean after committing
- the README or notes explain what to do
- your reflection names at least one problem and how you handled it

## Practice

Prepare your Track 0 project submission.

It should include:

- the project files
- a short README or notes file
- at least three meaningful commits, matching the Track 0 gate
- a debugging note
- a reflection

## Spine prompts

### Self-evaluation

What evidence shows this submission is ready?

### Technical communication

Could someone else understand what you did without asking you first?

### AI collaboration

Confirm that no AI was used for Track 0 curriculum work.
