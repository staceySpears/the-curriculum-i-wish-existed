# Module 4 — Editor and Tooling

## Goal

Open a project in a code editor and understand the difference between editing files and running files.

## Core idea

A code editor helps you read and change files.

A terminal helps you run commands.

Many editors include a terminal, but editing and running are still different actions.

## What an editor gives you

A good editor helps you:

- see the project folder
- open multiple files
- search across files
- notice syntax highlighting
- use an integrated terminal
- track changed files

The editor does not automatically make code correct. It is a workspace, not a guarantee.
