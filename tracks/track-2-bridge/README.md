# Track 2 — The Bridge

Track 2 is the anti-fragmentation track.

Many learners can write code from scratch but cannot read, trace, or safely modify code they did not write. This track repairs that gap deliberately.

The Bridge is not about learning a new language or framework. It is about building the habit of understanding before changing.

## What this track teaches

- reading unfamiliar code without panicking
- finding entry points in an existing codebase
- tracing data flow across files and functions
- identifying responsibilities and boundaries
- reading tests to understand expected behavior
- making small, safe changes to existing code
- explaining code that someone else wrote
- using debugging tools to trace execution
- asking good questions about code before modifying it

## AI policy

AI is not allowed for Track 2 curriculum work.

Do not use AI to explain code for you, summarize files, generate changes, write annotations, or complete the project.

The goal of this track is to build the skill of reading and understanding code yourself. AI bypasses exactly the learning this track is designed to produce.

See `../../AI_POLICY.md`.

## Modules

Modules cover entry point identification, data flow tracing, reading tests, making safe changes, and explaining unfamiliar code to another person.

The Track 2 project involves reading and safely extending an unfamiliar codebase. You will trace, modify, test, and document a small but non-trivial system.

See `project/README.md`.

## Gate

The Track 2 gate proves that you can understand and modify code you did not write.

See `gate.md`.

## Track 2 definition of done

Track 2 is complete when you can:

- read unfamiliar code and explain what it does
- trace data flow across files without running the code first
- identify where a change should go and why
- make a small safe change without breaking existing behavior
- explain what the existing tests verify
- explain the purpose of each major file or module
- pass the Track 2 gate
