# Track 2 Exercises

These exercises are about reading, tracing, and documenting — not building from scratch.

## Exercise files

| File | What it teaches |
|---|---|
| `InventoryManagerTest.java` | Writing tests for existing code; 18 stubs, 6 expected failures |
| `TraceCallChain.md` | Written trace exercise across a call sequence; Part 4 introduces a bug |
| `DecisionNoteTemplate.md` | Format for documenting design decisions; includes a worked example |

## How to use each file

### InventoryManagerTest.java

1. Copy `legacy-codebase/InventoryManager.java` into your project's `src/` directory
2. Read `InventoryManagerTest.java` fully before writing any test implementations
3. Implement the stubs one by one in the order they appear
4. Tests marked "this test will FAIL on the buggy version" are expected to fail until you fix the implementation in Module 3

### TraceCallChain.md

Work through the four parts in order:
- **Parts 1–2**: trace on paper and predict outputs, no running code
- **Part 3**: run the code and compare to your predictions
- **Part 4**: introduce the bug, retrace, document how you would find it

Do not skip to Part 3 before completing the written trace.

### DecisionNoteTemplate.md

Use this template each time you make a nontrivial design decision during Track 2.
Minimum two decision notes are required for Module 8. You may write more.

The template includes a worked example — read it before writing your own notes.
