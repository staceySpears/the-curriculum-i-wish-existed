# Track 4 — Frontend Fundamentals

Track 4 teaches the browser as a platform.

This track introduces HTML, CSS, and JavaScript as a stack. The goal is not visual polish. The goal is functional, accessible, understandable interfaces — and the habit of asking what happens when something goes wrong or is missing.

## What this track teaches

- HTML structure and semantics
- CSS layout: box model, flexbox, basic responsive design
- JavaScript in the browser: DOM manipulation, events, state
- Form handling and input validation
- Accessibility basics: labels, focus, keyboard navigation, ARIA roles
- Edge-case thinking applied to UI: empty states, missing data, long strings, invalid input
- Debugging in the browser using developer tools
- Reading and modifying existing frontend code

## AI policy

AI may be used as scaffolding in Track 4.

AI may generate code, suggest implementations, help debug, and give feedback. Every line of generated code must be read, understood, and defensible before it is submitted.

Before committing AI-assisted code, ask:
> Could I rewrite this from scratch without AI right now?

If the answer is no, you need more time with the material.

See `../../AI_POLICY.md` and `../../spines/ai-collaboration.md`.

See `gate.md`.

## Track 4 definition of done

Track 4 is complete when you can:

- structure a page with semantic HTML
- apply CSS layout with flexbox
- write JavaScript that manipulates the DOM and handles events
- validate user input and show meaningful errors
- apply basic accessibility practices
- describe what your UI does when data is empty, missing, or invalid
- debug UI problems using browser developer tools
- pass the Track 4 gate
