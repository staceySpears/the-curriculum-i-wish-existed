# Track 5 — React and Vite

Track 5 is where the learner builds component-based frontend applications.

This track introduces React and Vite as a production-oriented development stack. The goal is to build applications where behavior is provable, not just visible.

## What this track teaches

- React component model: props, state, rendering
- Hooks: useState, useEffect, and custom hooks
- Component composition and data flow
- Vite for development and build
- Fetching data from an API and handling loading and error states
- Testing React components
- Managing side effects cleanly
- Refactoring and identifying component boundaries

## AI policy

AI may be used as scaffolding in Track 5.

AI may generate code, suggest implementations, propose tests, help debug, and give feedback. Every line of generated code must be read, understood, tested, and defensible before it is committed.

Before committing AI-assisted code, ask:
> Could I rewrite this from scratch without AI right now?

For team learners, material AI-generated code must be flagged in pull request descriptions.

See `../../AI_POLICY.md` and `../../spines/ai-collaboration.md`.

## Track 5 definition of done

Track 5 is complete when you can:

- build React components with props and state
- use hooks for state management and side effects
- fetch data from an API and handle loading, success, and error states
- write tests that verify component behavior
- explain how data flows through a component tree
- refactor a component without breaking its behavior
- pass the Track 5 gate
