# Track 7 — Engineering Practice and Modern Tooling

Track 7 teaches the practices that make someone easier to trust as an engineer.

This track is not about learning new syntax or frameworks. It is about the habits of professional software engineering: code review, observability, CI/CD, safe refactoring, and collaborative workflow.

## What this track teaches

- reading and writing useful code review feedback
- using logs and metrics to understand system behavior
- CI/CD basics: build, test, and deploy pipelines
- refactoring safely: changing code without breaking behavior
- working in a shared codebase: branches, PRs, merge conflicts
- writing documentation that ages well
- using AI as a professional collaborator while maintaining judgment
- observability: what to watch, when to alert, how to investigate

## AI policy

AI may be used as a professional collaborator in Track 7.

Use AI to brainstorm, compare tradeoffs, draft documentation, generate test cases, inspect errors, and accelerate routine work.

The standard rises here. If AI introduces a bug, security issue, hallucinated API, or weak test — you are responsible for catching it. Professional AI use is supervised delegation, not passive acceptance.

See `../../AI_POLICY.md` and `../../spines/ai-collaboration.md`.

## Track 7 definition of done

Track 7 is complete when you can:

- write specific, useful code review feedback
- add logging and explain what it tells you about system behavior
- refactor code without breaking behavior
- describe what each stage of a CI/CD pipeline does
- use AI as a collaborator while catching its mistakes
- write documentation that a future maintainer can use
- pass the Track 7 gate
