# Capstone Gate

This gate verifies that you are ready to complete the curriculum.

The goal is not perfection. The goal is ownership: you can choose, shape, finish, and defend a project on your own terms.

## Required artifact

Submit your capstone project plus:

- a deployed or clearly runnable project
- a project README
- a project walkthrough (recorded or written)
- a verification strategy showing the project works
- a self-evaluation
- a debugging or revision note

## Correctness requirements

You must be able to:

- run or access the project without guidance
- demonstrate that it behaves as described in the README
- show passing tests, a manual test plan, or another concrete verification approach
- explain what is and is not handled

## Explanation requirements

You must explain:

- what problem the project solves and who it is for
- the major architectural decisions and why you made them
- at least one tradeoff and what you gave up
- what you would improve with more time
- where the project is most fragile and why
- how you would debug a failure if it happened in front of a user

## Debugging and revision evidence

Provide at least one debugging note that includes:

- what you expected
- what happened instead
- what you checked first
- what hypothesis you tested
- what fixed it or what you learned

Also show one instance of revision: a test added, a feature scoped down, a design changed based on feedback, or a behavior caught by testing that you then fixed.

## Transfer requirement

Show that the skill transfers. Options:

- explain how you would extend the project in a new direction
- identify what would change if the user base or data scale increased
- compare the architecture choice you made against one you considered and rejected

## Self-evaluation

Complete a self-evaluation answering:

1. What works well?
2. What is still fragile?
3. What was hardest?
4. What did you debug or revise?
5. What tradeoff did you make?
6. What would you improve with more time?
7. What evidence shows you are ready to complete the curriculum?

## AI disclosure

If AI materially assisted the project, describe:

- what you asked AI to help with
- what output you used
- what you changed or rejected
- how you verified the result
- what you now understand independently

If AI was not used, write: "No material AI assistance used."

## Spine evidence

### Big-O and data structure choice

- What structure, algorithm, or design choice mattered in your project?
- Why did it fit the problem?
- What tradeoff did it make?

### Discipline of Doubt

- What assumption, edge case, or failure mode did you consider?
- How did that change your implementation?

### Debugging

- What broke, confused you, or required investigation?
- How did you reproduce, observe, hypothesize, test, and revise?

### Code reading

- What existing code, library, framework, or unfamiliar pattern did you need to understand?
- How did you trace it before using or modifying it?

### Testing maturity

- What behavior is covered by tests or verification?
- What important behavior is not yet covered?
- What gave you the most confidence that the project works?

### Self-evaluation

- What is solid?
- What is still fragile?
- Why is this ready to present as a finished project?

### Technical communication

- How did you make the project understandable to a reviewer, future maintainer, or interviewer?
- Does your README let a stranger run and understand the project without asking you questions?

### AI collaboration

- Was AI used materially?
- If yes, how did you verify and maintain ownership?

## Solo learner add-ons

Solo learners should also include at least one of:

- recorded walkthrough
- comparison against a public project in a similar space
- external review from a peer, mentor, or community member
- expanded reflection covering what you would do differently
- extra test plan or edge case documentation

## Team learner add-ons

Team learners should also include:

- task or issue breakdown
- pull request history
- code review evidence
- pairing or collaboration notes
- individual reflection from each teammate
- retrospective summary
