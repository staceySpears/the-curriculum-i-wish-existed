# Capstone - Your Project, Your Terms

## Purpose

Bring the curriculum together in a substantial self-directed project.

## What this track teaches

Problem definition, scope, tool choice, tradeoffs, iterative build, testing, deployment, documentation, feedback, presentation, and reflection.

## What learners should already know

All prior tracks, especially backend, frontend, testing, deployment, interview storytelling, and engineering practice.

## How the track works

Each track uses the same basic rhythm: learn the concept, practice it in small pieces, integrate it in a project, then pass a gate that asks for evidence. Modules should introduce concepts before tools, name the assumptions learners need, and give enough practice that understanding can be used without a script.

## Project

A full-stack or otherwise substantial project with a clear user, real functionality, tests, docs, deployment, and presentation.

## Gate

The learner can build, test, deploy, present, defend tradeoffs, and show revision from feedback or use.

## Spine integration

Every track should produce evidence across the eight spines: data-structure reasoning, Discipline of Doubt, debugging, code reading, testing maturity, self-evaluation, technical communication, and AI collaboration.

## AI use

AI may be used as a professional collaborator, with full learner ownership.
