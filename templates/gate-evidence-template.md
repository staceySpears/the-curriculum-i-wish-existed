# Gate Evidence Template

Use this to collect evidence before attempting a gate.

## Before you start

- [ ] I have completed the required modules
- [ ] I have completed the project
- [ ] I have run all tests or verification checks
- [ ] I have written or updated the README

## Spine evidence

### Big-O / data structure
- [ ] I can explain the time complexity of my main operation
- [ ] I can justify my data structure choice

### Discipline of Doubt
- [ ] I considered at least two edge cases
- [ ] I documented what assumptions I made

### Debugging
- [ ] I encountered at least one issue
- [ ] I documented: what I observed → what I hypothesized → what I tried → what fixed it

### Code reading
- [ ] I read existing code before writing new code
- [ ] I can trace the entry point to the relevant feature

### Testing maturity
- [ ] I wrote tests (or manual verification steps)
- [ ] I have at least one test for a success case and one for a failure case

### Self-evaluation
- [ ] I know what works
- [ ] I know what is still fragile
- [ ] I know what I would improve with more time

### Technical communication
- [ ] My README explains what this is, why it exists, and how to run it
- [ ] Someone else could understand my work

### AI collaboration
- [ ] I documented any material AI assistance
- [ ] I can explain, verify, and modify any AI-influenced code

## Ready for gate?
- [ ] I have all required artifacts
- [ ] I can explain my work without AI
- [ ] I can debug or modify it without AI
- [ ] I am ready for the gate conversation
