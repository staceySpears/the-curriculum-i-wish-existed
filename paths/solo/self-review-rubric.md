# Solo Self-Review Rubric

Use this rubric before submitting a project, attempting a gate, or asking another person for review.

The goal is not to prove the work is perfect. The goal is to decide whether it is ready for the current stage.

## Correctness

Can I run the project or exercise successfully?

Do the provided tests pass?

Did I add tests or checks where appropriate?

Can I explain what behavior is covered and what is not covered?

## Understanding

Can I explain the important parts without reading from AI or copying the README?

Can I trace the main flow from input to output?

Can I explain one tradeoff I made?

Can I modify one small part without starting over?

## Debugging evidence

Did something break while I worked?

Can I explain how I investigated it?

Did I preserve any useful debugging notes, tests, or commits?

## Spine evidence

Can I explain any relevant data structure choice?

Can I name an edge case or assumption?

Can I describe how I read or traced existing code?

Can I explain what tests or checks give me confidence?

Can I communicate the work clearly to a future maintainer?

## Readiness decision

Choose one:

### Ready

The work meets the current standard. Weaknesses are known and do not block moving forward.

### Revise

The work is close, but one or more issues should be fixed first.

### Not ready

The foundation is unstable. Revisit the module or project before continuing.

## Final prompt

If someone reviewed this tomorrow, what would I want them to look at first?
