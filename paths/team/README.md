# Team Path

The team path is for learners working with the same small group throughout part or all of the curriculum.

A healthy team can accelerate learning. It creates feedback, accountability, shared debugging, code review, and practice communicating like engineers.

An unhealthy team can hide gaps, create resentment, let one person carry the work, or teach people to avoid conflict.

The team path exists to make collaboration intentional.

## Recommended team size

Two to four learners is ideal.

Two people can pair deeply.

Three people can distribute work while still staying aligned.

Four people can work well if communication is strong.

More than four should split into smaller teams.

## Team operating rules

Write a team charter before starting shared work.

Use issues or tasks so work is visible.

Use branches and pull requests for meaningful changes.

Review code before merging.

Rotate responsibilities.

Hold retrospectives.

Address anti-patterns early.
