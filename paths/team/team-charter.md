## Definition of done

For shared work, done means:

- code works locally
- checks pass where available
- README or notes are updated where needed
- pull request is reviewed
- relevant edge cases are considered
- task is moved to done only after merge or accepted review

Adjust this list for the track.

## Code review agreement

The team agrees to:

- review the code, not attack the person
- ask questions before assuming intent
- leave specific comments
- explain requested changes
- respond to review without defensiveness
- avoid merging meaningful work without review

## Conflict plan

When conflict appears, the team will:

1. Name the behavior or issue specifically.
2. Return to the charter.
3. Discuss impact without personal attacks.
4. Agree on the next behavior change.
5. Revisit during the retrospective.
